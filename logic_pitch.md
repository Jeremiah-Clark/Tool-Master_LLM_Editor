# Creative Pitch & Proposal Module

*For sales, design docs, and creative proposals.*

------

## 1. The "Hook" & Stakes

- **Immediate Value:** The "Elevator Pitch" must be visible without scrolling. 
- **High Stakes:** Use active, evocative verbs to describe the user experience (e.g., "Escape the shadow" vs. "The player runs").

## 2. Competitive Audit (Strategic Layer)
- **The "Why Now?":** Flag if the pitch fails to address the current market or cultural moment.
- **Differentiation:** If the project resembles an existing product (e.g., *Temple Run*), the document must answer: "How does this specifically differ from [Competitor]?"
- **Feature Prioritization:** Distinguish between "Core Loop" (must-haves) and "Flavor" (nice-to-haves).

## 3. Revenue & Platform Logic

- **Technical Feasibility:** Ensure creative descriptions don't hand-wave technical implementation. If a feature is "Procedural," the pitch must briefly explain the logic (e.g., "Tile-based" vs. "Granular").
- **Monetization Audit:** Flag contradictions between the "Core Loop" and the "Revenue Model" (e.g., selling power-ups in a "pure survival" game).
- **Platform Reach:** Ensure the "Control Scheme" matches the "Target Platform" (e.g., avoid complex combos in a mobile-first touch game).

## 4. Readability

- **Wall of Text Alert:** Flag any paragraph over 5 lines. Suggest breaking into bulleted lists or sub-headers to improve readability for busy stakeholders.
- **Tone:** Balanced between "Infectious Enthusiasm" and "Grounded Professionalism," and aligned with the intended market segment.
